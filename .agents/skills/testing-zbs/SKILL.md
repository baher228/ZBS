# Testing ZBS Agent Platform

## Starting the App

### Backend
```bash
cd /home/ubuntu/repos/ZBS/backend
source /home/ubuntu/repos/ZBS/.env  # loads PYDANTIC_AI_GATEWAY_API_KEY, FAL_API_KEY
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### Frontend
```bash
cd /home/ubuntu/repos/ZBS/frontend
npm run dev
# Serves on http://localhost:8080 (Vite config overrides default 5173)
```

## Content Generation Testing
- Content generation with iterative refinement takes ~60 seconds (generate + review + revise + re-review + image gen)
- Expects "Iteratively refined" badge in the review summary bar (checks if summary contains "revised")
- Review scores should be 55-90% range (strict calibration), NOT 100%
- Exactly 2 images generated: landing_copy and social_post (IMAGE_SECTIONS constant)
- Images come from fal.ai Flux Pro v1.1 (URLs start with `https://v3b.fal.media/`)
- Company context (e.g., "TestCorp AI") should appear in generated content

## Environment Variables

- Both stored in `/home/ubuntu/repos/ZBS/.env`

## Social Media Section on Onboarding
- Located below "Operating Jurisdictions" section
- 4 platform inputs: LinkedIn, Twitter/X, Instagram, Facebook
- Saved as `social_media_links` dict in CompanyProfile model
- Social insights extraction scrapes URLs and analyzes with LLM

## Known Behaviors
- Social media scraping with fake URLs gracefully degrades (returns empty insights)
- Image prompts are generated by gpt-4o-mini based on actual content text (contextual, not templates)
- Review Agent always provides revision_instruction for first drafts
- MIN_REVISION_ROUNDS=1 guarantees at least 1 revision before presenting results
