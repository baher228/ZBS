# Project Rules

1. Don't break frontend styling.
2. Preserve existing user or teammate work; inspect before editing and avoid unrelated rewrites.
3. Keep agent APIs structured with Pydantic-compatible request and response shapes.
4. Legal output must be educational, source-grounded, and clearly not legal advice.
5. Use official or primary sources for regulatory seed documents whenever practical.
6. Add focused tests when adding or changing backend agent behavior.
